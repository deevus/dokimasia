# uv Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate Dokimasia's development workflow from direct pip commands to uv with a committed lockfile and verified uv-based test command.

**Architecture:** Keep the existing setuptools-based package metadata in `pyproject.toml`; use uv only for environment synchronization, dependency locking, and command execution. The migration is documentation and lockfile focused, with verification through the existing pytest suite.

**Tech Stack:** Python 3.11+, setuptools, uv, pytest.

---

## File Structure

- Modify: `README.md`
  - Replace the pip editable-install instructions with uv-native setup and test commands.
  - Preserve the existing project description and Python usage example.
- Create: `uv.lock`
  - Generated by `uv lock` or `uv sync`; committed to make local development and CI-style test runs reproducible.
- No source files should change.
- No test files should change unless uv exposes a real test failure unrelated to installation.

### Task 1: Generate uv Lockfile

**Files:**
- Create: `uv.lock`
- Read: `pyproject.toml`

- [ ] **Step 1: Confirm uv is available**

Run:

```bash
uv --version
```

Expected: command exits 0 and prints a uv version, for example:

```text
uv 0.x.y
```

- [ ] **Step 2: Generate the lockfile**

Run:

```bash
uv lock
```

Expected: command exits 0 and creates `uv.lock` at the repository root.

- [ ] **Step 3: Inspect lockfile status**

Run:

```bash
git status --short uv.lock
```

Expected:

```text
?? uv.lock
```

or, if a lockfile was already generated before this step:

```text
 M uv.lock
```

- [ ] **Step 4: Verify uv can sync the project**

Run:

```bash
uv sync
```

Expected: command exits 0 and creates/updates the local uv environment. It should install the editable project and runtime dependencies.

- [ ] **Step 5: Commit the lockfile**

Run:

```bash
git add uv.lock
git commit -m "chore: add uv lockfile"
```

Expected: commit succeeds with `uv.lock` included.
